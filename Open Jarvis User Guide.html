Open Jarvis — How to run it, use it, and grow it

Open Jarvis turns your projects into marketing: it ingests a repo, understands the product, plans LinkedIn/TikTok content, drafts approval-based HTML storyboards & landing pages, renders videos, and researches competitors — all driven by your Gemini Webapp license through a managed Edge browser. No paid API key.

1 · What you're running

There are two layers. You mostly drive the bottom one — the browser module — and it feeds the top one.

Open Jarvis core

The local-first AI framework + your project-to-marketing workflow: ingest → interpret → plan → draft → approve → produce → review. Stores everything as HTML and files in a per-project workspace.

Playwright MDM module — the engine

tools/playwright-mdm/. Drives your signed-in Edge profile via CDP and uses the Gemini web app as the brain. This is what you click into and set up once.

The core idea — "Powered-By-Webapp-LLM"

Instead of paying per token for a Gemini API key, Open Jarvis opens gemini.google.com inside the Edge profile you're already signed into and reads its answers. Your existing Gemini subscription does the reasoning; Open Jarvis orchestrates and stores the output.

2 · Tech stack

Core (Open Jarvis)

  • Python ≥ 3.10, < 3.14 · Apache-2.0
  • Build: hatchling, packaged on PyPI
  • CLI: click · UI: rich
  • HTTP: httpx · LLM SDK: openai
  • Data: datasets · Search: ddgs
  • Channels: python-telegram-bot
  • Telemetry: posthog · GPU: nvidia-ml-py
  • Desktop app: Tauri (desktop/src-tauri)
  • Local models via Ollama

Browser module

  • Node.js ≥ 18 (tested on 22), ESM
  • Playwright ^1.60 (Chromium/Edge over CDP)
  • Microsoft Edge + remote debugging (:9222)
  • Brain: Gemini web app (your license)
  • Built-in node:http agent UI (port 3000)
  • Tests: node --test
  • Config: jarvis-mdm.config.json (remembered)

Output / production

  • HTML + Tailwind for storyboards, landing pages, previews & reports
  • Remotion (React) for video rendering
  • Per-project file workspace (markdown + HTML)
  • Shared publicity queue for cross-project assets

3 · Prerequisites (install once)

  • Node.js 18+ — check with node --version
  • Microsoft Edge, with at least one profile signed into your Gemini license
  • Python 3.10–3.13 (only needed for the Open Jarvis core, not the browser module)
  • For video: Remotion deps when you reach the render step
# one-time install of the browser module
cd "tools/playwright-mdm"
npm install
npx playwright install

4 · One-time setup — pick your Edge + brain, it remembers forever

Run this once. You choose which Edge profile to drive and which webapp powers Open Jarvis. The choice is saved to jarvis-mdm.config.json — from then on the bridge auto-launches that exact profile. You never pick again. 

cd "tools/playwright-mdm"
npm run setup

5 · Running Open Jarvis

After setup — just call the brain

Close Edge first (so the module owns a clean debug session), then:

cd "tools/playwright-mdm"

# the bridge AUTO-LAUNCHES your remembered
# profile, then asks Gemini:
npm run bridge -- "In one sentence, \
  confirm you are reachable."

No profile prompt — it reads jarvis-mdm.config.json and launches Edge for you.

Other entry points

  • npm run gemini — interactive CLI chat through your license
  • npm run gemini-web — local agent UI at http://localhost:3000
  • npm start — attach + smoke-test an app URL
  • npm test — run the module's unit tests

Calling it from Open Jarvis code

The core never touches Playwright — it calls three functions:

import { askGemini, research, act, dispose } from './tools/playwright-mdm/jarvis-bridge.js';

// REASON — produce content with your Gemini license
const summary = await askGemini('Summarise this repo for a LinkedIn launch post: ...');

// RESEARCH — look at a live page on a SEPARATE tab, analyse with Gemini
const findings = await research(
  'https://www.linkedin.com/in//recent-activity/',
  'Classify recent post types and infer their objectives',
);

// ACT — let Gemini plan browser commands, the agent runs them
const { plan, results } = await act('Open the product page and extract the headline');

await dispose(); // on shutdown (leaves Edge running)

Gemini runs in a dedicated tab that's never navigated away — research uses a separate page, so no clobbering. Every call is logged to reports/gemini-sessions/.

6 · Using it for your goal (project → marketing)

This maps the seven-stage workflow from your Architecture Plan onto concrete bridge calls. The approval gate stays human — nothing publishes without you.

1 · Ingest

Drop the repo/notes/raw HTML into workspace/projects/<name>/source/.

2 · Interpret

askGemini("Read this repo summary and write project-summary, goals, audience, challenges") → save to understanding/.

3 · Plan

askGemini("Propose 5 LinkedIn + 5 TikTok ideas for this product") → marketing/*/ideas/.

4 · Draft

askGemini writes storyboard HTML, scripts, and landing-page HTML → drafts/, landing-page/html/, video/storyboard/.

5 · Approve

Open the draft HTML in a browser, review, move from approvals/pending/ → approved/. Human gate — required.

6 · Produce

Render approved storyboards with Remotion → video/renders/; finalise landing pages.

7 · Review

research() monitors LinkedIn/competitors → research/observations/; feed learnings back into the next interpret pass.

7 · Expanding it further

Add another webapp brain

Add an entry to LLM_PRESETS in config.js (name + url). For a non-Gemini DOM, copy gemini.js to a new adapter and update the SEL selectors (input / send / stop / response). Re-run npm run setup and pick it.

Add new browser actions

Extend executeCommand() in playwright-agent.js with new action cases (e.g. hover, selectOption, download). They become instantly available to act().

Wire core stages to the bridge

Call askGemini/research from the Interpret and Draft stages of Open Jarvis core so the whole pipeline runs on your license. Keep storing transcripts in gemini-sessions/ for the learning loop.

Harden it

Add a "DOM drift" smoke check that fails fast if selectors change; add throttling/retries; multi-profile presets (e.g. one config per client) by parametrising the config path.

8 · Troubleshooting & guardrails

  • "Cannot attach / it hangs": your everyday Edge has many tabs/profiles (Playwright stalls attaching to all of them). Fully close Edge, then run the bridge — the module launches a clean single-profile debug session it can attach to.
  • "No Edge profile remembered": run npm run setup first.
  • Port 9222 busy: some other Edge/automation owns it. Close all Edge windows and retry.
  • Selectors broke: Gemini changed its DOM — update SEL in gemini.js.
  • Sign-in: stays manual in the Edge profile; the module never automates Microsoft/Google login.
  • Compliance: automating a web app is governed by Google's ToS and your org's MDM policy. Keep this single-user and human-supervised; the approval gate means nothing auto-publishes.
