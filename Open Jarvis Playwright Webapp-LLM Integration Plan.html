Integration Plan Powered-By-Webapp-LLM

Open Jarvis × Playwright MDM Browser — Webapp-LLM Integration Plan

This plan extends the existing Open Jarvis Architecture Plan by adding a Playwright research & automation module that drives Microsoft Edge through your managed (MDM) browser profile. Crucially, the module does not consume a paid Gemini API key — it drives the Gemini web app inside your already-signed-in Edge profile, so Open Jarvis uses your Gemini Webapp license as the reasoning brain that decides what marketing content to produce.

Source A — Runner shell

raid-ppcoe/Playwright-Module-for-MDM-Browsers

An agent-oriented e2e-runner template. Its README is a prompt that asks the coding agent to import the code, pick an Edge profile, enter an app URL, and loop test → fix → HTML report → build/push until the app is bug-free.

Source B — Engine

ElfredSeow/Playwright-for-MDM-Browser

attach.js handles Edge profile discovery, CDP attach (127.0.0.1:9222), navigation, and smoke-test reporting. Reuses the signed-in MDM profile, so org sign-in stays manual/profile-based.

Source C — The brain

branch Powered-By-Webapp-LLM

gemini.js + gemini-server.js + playwright-agent.js drive the Gemini web app in Edge, read its replies, parse JSON command blocks, and execute them. This is the license-powered LLM Open Jarvis will call.

The key idea — "Powered-By-Webapp-LLM"

Open Jarvis normally needs an LLM to interpret a project, plan marketing, and write storyboards/scripts/landing copy. Instead of paying per-token for a Gemini API key, the Webapp-LLM adapter sends those prompts to gemini.google.com through your authenticated Edge profile and scrapes the answer. Your existing Gemini subscription does the thinking; Open Jarvis just orchestrates and stores the output.

Two ways Jarvis can call it

  • Reason mode — ask Gemini to produce content (summaries, ideas, scripts, copy). Jarvis stores the text.
  • Act mode — Gemini returns JSON command blocks (navigate, click, type, extract, screenshot…) that the Playwright agent runs to research live pages.

Why it fits Open Jarvis

  • No API key or per-token billing — uses the seat you already own.
  • MDM/org sign-in is respected (cookies live in the Edge profile).
  • Same browser session can both research LinkedIn/competitors and generate the marketing artifacts.

⚠️ Compliance note: scraping a web app is governed by Gemini/Google ToS and your org's MDM policy. Keep this as a single-user, human-supervised assistant with throttling and an approval gate — do not run it as an unattended bulk service.

The bridge — how Open Jarvis core talks to the module

A small jarvis-bridge.js exposes two functions to the rest of Open Jarvis. The core never touches Playwright directly — it just asks for reasoning or research and gets text/JSON back, keeping the approval workflow intact. 

// tools/playwright-mdm/jarvis-bridge.js  (sketch)
import { attachToEdge, getAttachedBrowserContext } from './attach.js';
import { navigateToGemini, sendPrompt, waitForResponseComplete,
         extractLatestResponse } from './gemini.js';

// REASON: use the Gemini Webapp license to PRODUCE content
export async function askGemini(prompt) {
  const ctx  = await getAttachedBrowserContext();   // signed-in MDM Edge
  const page = await ctx.newPage();
  await navigateToGemini(page);
  await sendPrompt(page, prompt);
  await waitForResponseComplete(page);
  return extractLatestResponse(page);               // → text for Jarvis to store
}

// ACT: research a live page via Gemini-planned commands
export async function research(url, objective) {
  return askGemini(`Visit ${url}. Goal: ${objective}.
Reply ONLY with JSON command blocks the Playwright agent can run.`);
}

Integration build order

  1. Clone the module into tools/playwright-mdm/ from raid-ppcoe/Playwright-Module-for-MDM-Browsers.
  2. Overlay the Webapp-LLM branch files (gemini.js, gemini-server.js, playwright-agent.js) from ElfredSeow/Playwright-for-MDM-Browser @ Powered-By-Webapp-LLM.
  3. npm install & npx playwright install; verify npm test passes.
  4. Pick the MDM Edge profile once; confirm CDP attach on 127.0.0.1:9222 and a Gemini smoke prompt round-trips.
  5. Write jarvis-bridge.js (askGemini / research).
  6. Wire Interpret + Draft stages to call askGemini.
  7. Wire LinkedIn research to call research; save transcripts to gemini-sessions/.
  8. Keep the approval gate before any produce/publish step.

  • Store findings → research/observations/ + gemini-sessions/

Guardrails & open questions

  • Fragility: the adapter targets Gemini's DOM selectors — they change. Keep selectors in one config and add a "DOM drift" smoke check.
  • ToS / MDM policy: confirm web-app automation is permitted on your managed device; keep it single-user and human-supervised.
  • Rate & reliability: throttle requests, retry on stop-button timeouts, and never auto-publish without the approval gate.
  • Auth: sign-in stays manual in the Edge profile; the module never automates Microsoft/Google login.
  • Resolved & implemented: Gemini runs in a dedicated tab that is created once and never navigated away; research() uses a separate page for the target URL, so there is no navigation clobbering.
