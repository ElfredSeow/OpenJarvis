Open Jarvis HTML Architecture Plan

Open Jarvis is a project-to-marketing automation system that ingests your repos, understands the product, creates approval-based storyboard HTML, generates Remotion videos, builds landing pages, and stores all publicity work in a shared queue.

Core operating model

  1. Ingest repo, notes, raw HTML, and research inputs
  2. Interpret project context, goals, audience, and challenges
  3. Plan LinkedIn and TikTok opportunities
  4. Draft storyboard HTML, scripts, and landing pages
  5. Approve before production
  6. Produce videos and pages
  7. Review and improve future outputs

Approval workflow

1. Ingest
2. Interpret
3. Draft
4. Approve
5. Produce
6. Review

Storyboard HTML

One HTML file per scene or screen, including title, intent, on-screen text, voiceover, CTA, and transition notes.

Landing page HTML

Generate a previewable landing page, test it, then iterate before finalizing.

Marketing preview HTML

Preview LinkedIn and TikTok content in HTML before exporting or rendering.

Playwright MCP research module

  • Monitor selected LinkedIn profiles
  • Classify post types and recurring themes
  • Infer objectives behind posts
  • Store findings in the research folder

Recommended build order

  1. Define folder structure
  2. Build project ingestion and summarization
  3. Add storyboard HTML approval workflow
  4. Connect Remotion rendering
  5. Add landing page generation and testing
  6. Add LinkedIn research monitoring
  7. Add shared publicity queue
  8. Add learning from historical HTML
